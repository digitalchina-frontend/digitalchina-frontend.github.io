# 严格相等比较运算符

更好的使用严格相等比较运算符

## ==（不推荐）

```js
1 == '1' // true
0 == '' // true
null == undefined // true
[] == '' // true
// ==只比较值，不比较类型，一般情况下会埋下隐患，特殊情况下可以使用
```

## ===（推荐）

```js
1 === '1' // false
0 === '' // false
null === undefined // false
[] === '' // false
// ===既比较值，也比较类型，一般情况下会埋下隐患
```

## 问题

任何情况下都应该优先使用===吗
